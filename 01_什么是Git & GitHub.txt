Git 是什么？

Git实际上被称为版本管理，当你在手动创建一些文件的时候，你会发现在你日期更新时，你就不得不重命名以及新建一个文件，以用来保证其更新效果。
而Git就是能够让你在不重建文件的前提下，一个文档或者你输入的任何文档，进行对应的版本更新迭代。并且能够让你迭代和更新不同的文档。

它还有一个功能就是用于合作，合作是一件非常必要的事情，Git是一个高级合作工具，能够让多个人在不同的时间，不同的位置合作出一个相同的东西。

它是一个系统，能够让和你一起工作的人，能够看到这个文档进行工作，并且能够让你看到这些文档的历史状态，以及相关的历史更新说明。这个相对于你平时的单个文件的手动更新，可能来的更为直观一些。

Git是一个版本工具软件，它是一个被赞助的软件，Git是一个版本控制软件，但不是唯一的一个版本控制软件，你也可以有其他的版本控制软件来对你的文档进行版本的控制。

GitHub
GitHub是一个在线的网络服务，可以让你在Github的远程网络中进行本地化的版本控制操作。有人会称之为"社交代码"工作,它是一个类似于用于项目的社交网络，可以让你管理这些工具。

在这里，你需要明白Git和GitHub是分开的东西。Git是一个版本控制的软件，GitHub是一个在线的网络服务。这两个东西都可以分别独立的运行。

Repository是什么意思？ 
Repository是传统意义上面说的仓库，它能够存储你的文档，你的文件，你创建了你的Repository你就等于创建了一个文档库。就像我当前的这个文档库叫做《GitHub Beginner Tutorial》，它能够存储各种各样的文档。

Commit 是什么意思？
Commit的意思很简单，相当于在说"存储并对当前文档进行说明"，在这其中包含两个含义。一方面在说存储，另一方面在说文档的说明。这就是它不一样的地方，因为每当你存储的时候，Git或GitHub都会希望你对你当前存储的版本做一个详细的解释。

当你对文件进行了Commit操作之后，你就能看见当前Commit的行为历史记录，例如你在文档里面增删改查了哪些内容。
